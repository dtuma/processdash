Inspection Script

Phase
Number		 Purpose To guide you in recording an inspection for an SCR or SCR Cycle.
Inputs Required An SCR, or SCR Cycle ready for inspection.
An Inspection Report form.
1 Planning Enter planned times into the Inspection Summary.
2* Design Inspection Using the Design Inspection Checklist, inspect the following (as applicable):
  • SRS,
  • SDD,
  • Interface Control Documents (ICDs),
  • Design Inspection sections of the SCR,
  • Test plans, outlines and operational scenarios,
  • Estimates.
Log all defects found and time spent on the Inspection Report form attached to the design.
Inform the producer when inspection is complete.
Record your time on the dashboard.

* This phase should only be performed if inspecting a design. If inspecting code, perform the Code Inspection phase instead.
3* Code Inspection Using the Code Inspection Checklist, inspect the following
  • Code changes,
  • Code Inspection sections of the SCR,
  • SUM changes,
  • SDD changes, and
  • any other supporting documentation changes.
Log all defects found, and time spent on the Inspection Report form attached to the code.
Inform the producer when inspection is complete.
Record your time on the dashboard.

* This phase should only be performed if inspecting code. If inspecting a design, perform the Design Inspection phase instead.
4 Postmortem Check the completion box on the Dashboard.
  Exit Criteria
  • A thoroughly inspected design / code.
  • Completed Inspection Report form.
