1         The Dashboard GUI

The dashboard GUI consists of a minimal frame surrounding the following items:

  • The Configuration Menu
  • The Play/Pause Button
  • The Defect Button
  • The Script Button
  • The Hierarchy Menus
  • The Completion Checkbox

The dashboard resizes itself (when button layout changes) to maintain as small a profile as possible. The term "hierarchy" is used throughout this document to refer to the work-breakdown structure defined in the Hierarchy Editor and navigated with the Hierarchy Menus.

1.1        Configuration Menu

The Configuration Menu appears as a "C" on the dashboard, and contains several important options that allow you to edit your work breakdown structure or your time and defect logs.  Items in this menu also provide access to a PROBE tool, an import/export tool, and this help document.

1.1.1       Hierarchy Editor

The Hierarchy Editor window has a tree view of the work-breakdown structure nodes / tasks defined on the dashboard. Node names (excluding most template sub-nodes) may be edited by triple clicking on the node name in the tree view window.

The File menu allows the user to Save, Revert, or Close the window. Closing the window does not cause unsaved changes to be lost. The changes are lost ONLY if "Revert" is selected, or if the dashboard is closed before "Save" has been selected.

The Edit menu options allow the user to Delete nodes, Add Nodes (above the currently selected node, below the currently selected node, or as a child of the currently selected node), and Add Templates. The availability of these options is dependent upon the node currently selected. For example, when a template node or sub-node is selected, neither nodes nor templates may be added. See section 2 (Templates) for information on exceptions to this.

The Add Templates option is a submenu containing defined processes (e.g. PSP2.1) that you can use to perform a task.  If this submenu is empty, see the Troubleshooting section at the end of this document.

1.1.2       Time Log

The Time Log window has a set of filter controls, a tree view of the nodes in the hierarchy, a table view of the current time log entries, and a set of control buttons.

1.1.2.1       Filter Controls

Edit the "from" and / or the "to" date(s) and click on the "Apply Filter" button to apply the date filter to the tree view and to the table view.

1.1.2.2       Tree View

The time spent on a task / collection of tasks is displayed to the right of the displayed name (this time includes a rollup of any time assigned to child nodes). The displayed time is also subject to the time filter applied by the Filter Controls. Selecting a node filters the Table View to display only the time entries assigned to that node and its children.

1.1.2.3       Table View

The time entries are displayed in table format. These entries are editable, but changes are not permanently saved until you click the "Apply" button. Delta and Interrupt times are displayed in "HH:MM" format. The "Logged To" column must equate to an existing node name to be accepted. To change time and date display formats, see section 5.

1.1.2.4       Control Buttons

The Add button adds a new time entry field for the user to edit / complete. The Delete button deletes the currently selected log entry. The Reload button causes the time log to revert to its state as of the last time it was saved. The Apply button saves the (edited) time log.  (Note: if the Apply button is never pressed, any outstanding changes will currently be lost when the dashboard is shut down.)

1.1.3       Defect Editor

The Defect Editor allows you to edit defects and uses a layout similar to the time log editor.

1.1.4       PROBE

Performs correlation for making estimate projections based on existing historical data. Typical entries for the "Correlate" and "with" fields are "Estimated LOC" and "LOC". Then enter the estimate and range to get the projection, and upper and lower prediction intervals. Colored parameters indicate that linear regression estimates may not be appropriate (use with caution).

·         Use the "Correlate" and "with" fields to enter the data pair (x and y) you want to correlate - some of the common pairs are displayed in the table below.  There are many other pairs - use whatever works for you.

·         Use the "Estimate" field to enter the estimated value (corresponding to the "Correlate" field).

·         Use the "% range" field to enter the confidence value desired (e.g., "70" or "90" to get range / UPI / LPI data).  These four fields are required.

·         There are two columns beneath the data fields.  The left hand column provides a linear regression solution to the estimate.  The r-squared (correlation) and p (significance - percent chance that correlation is accidental) values are color coded to help indicate how reliable the estimate is.  The colors are: red (very bad), yellow/orange (bad), blue (good), and black (very good).

·         The right hand column is the basic average estimate, corresponding to PROBE method C (Beta0 = 0).  Use this column if the results in the left hand column are suspect.

·         By default, PROBE filters out projects which have not been marked complete with the completion checkbox.  To change this behavior, see section 5.

·         The "filter" button provides a method to exclude unfinished projects and/or outliers from the calculations.  This button brings up a dialog with a tree view of your current projects.  Check/uncheck projects/subprojects to include/exclude items from the calculations.  Press the "Apply" button to see the effect on the PROBE estimate.

·         The "Chart" button provides a scatter plot (and table list) of the data points, as well as the lines corresponding to both the "Linear Regression" and "Average" estimates.  The Chart and filter dialogs work well together to determine probable outliers.

·         Use the appropriate projection as your final, adjusted estimate.

1.2        Play/Pause Button

Press the play / pause button to start or stop recording time for the currently selected hierarchy. By default, the button graphic displays the action to be taken (e.g., the "Play" triangle is pressed to start recording), but this may be reversed by the user (see section 5 – Settings).

1.3        Defect Button

Press this button (indicated by a picture of a bug), to bring up a defect recording dialog. This dialog allows the selection of an error type, related fix defect, phase injected, phase removed, fix time, and a description of the defect. Select OK to save the defect into the defect log. The Stop Fixing / Start Fixing button acts as a timer. By default, the dialog comes up timing, so that the user may then fix the error, document the fix in the dialog and click OK (the time elapsed is automatically entered into the Fix Time box). To change the default, see section 5 – Settings.

1.4        Script Button

This button brings up the scripts and forms associated with the currently selected hierarchy node in an HTML browser. A right-click on this button provides a popup list of available scripts – the top entry in this list is the script that will be opened by a left-click.

New processes, scripts, and forms can be created for use with the dashboard; see section 2 (Templates) for information.

1.5        Hierarchy Menus

The hierarchy button(s) allow the user to select a hierarchy node. From left to right, these buttons track the hierarchy of the currently selected node.

1.6        Completion Checkbox

This checkbox tags the currently selected hierarchy node as completed (with the current date/time), and changes the currently selected node to the next node in the hierarchy.

2         Templates

A template consists of entries in the "state" file in the designated template directory (see section 5 – Settings for information on setting this directory), and (possibly) a collection of scripts and data files.

3         Scripts

A script is an HTML file that defines the process to be used for given template. These files may contain data (see section 4.3), instructions and links. All forms must put the "data.js" script at the bottom.

4         Data

Data is expressed as a name / value pair. The value may be expressed as a constant or as a formula. Available data formats are string, number, and date. As tabs are used as separators, data names may not contain tabs. Data is accessed via leaf name or by hierarchy + leaf name. A double slash (//) may be used in an html file to indicate a value or formula that should NOT be saved to the persistent data file(s).

4.3        HTML File

HTML scripts and forms can include active elements on the page which are tied to process data.  To accomplish this, simply

  • Create a <FORM> on the HTML page
  • In the form, add <INPUT> tags;  the NAME of the tag specifies which piece of process data should be tied to the input tag.
  • Add the following text to the bottom of the HTML page:
                    <SCRIPT SRC="/0/data.js"></SCRIPT>
    This "breathes life" into the <FORM> and causes the form elements to be tied to process data.

Look at the forms and scripts which come with the dashboard for examples.  The syntax below gives some additional information about the NAME attribute;  in particular, after specifying the name of the process data element, you can insert a tab character followed by flags which control the display of the data.

5         Settings

Settings may be made in several places. Defaults are set in the code to allow operation even if no setting files can be found. The dashboard then attempts to read a system level settings file (pspdash.ad). The dashboard then attempts to read the local settings file (pspdash.ini on Windows or .pspdash on other platforms). The dashboard searches for this settings file in the current directory, then in the user’s home directory (as specified by Java).  The following settings may appear in the settings file

Appearance of the pause / play button on the dashboard.

true — shows PLAY symbol while playing (current state)

false — shows PLAY symbol while paused (affect of button press)

probeDialog.onlyCompleted

implicit filtering performed by PROBE dialog

true – PROBE only correlates data from completed projects

http.allowRemote

allow or deny remote http connections

By default, the dashboard rejects http connections unless they come from "localhost".  This prevents other people from viewing your PSP data.  If you want to be able to view your own PSP data in a browser on another machine, set this to "true".

8         Troubleshooting

If the dashboard doesn’t start or if there are no entries in the Hierarchy Editor’s "Edit -> Add Template" menu, then JRE 1.3 is not completely setup on your machine.  To workaround this problem:

  1. Right click on the icon you use to run the dashboard, and choose "Properties"
  2. Click the "Shortcut" tab, and edit the "Target" field to read
    "C:\Program Files\JavaSoft\JRE\1.3\bin\javaw.exe" -jar "C:\Program Files\PSPDashboard\pspdash.jar"
    (of course, substitute the correct paths to your java runtime environment and to the dashboard jar file, as appropriate.)
